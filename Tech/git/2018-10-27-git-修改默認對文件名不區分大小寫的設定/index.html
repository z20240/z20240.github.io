# Git 修改默認對文件名不區分大小寫的設定

# 踩過地雷才會知道，哪裡會爆炸

前些日子在修改專案名稱時，發現將只單純修改大小寫並不會使 git 偵測到文件有被修改。 去網路上爬文才發現原來 git 本身對文件名是默認不區分大小寫的。 因此決定寫一篇文章將修改此設定的方法記錄下來。

前些日子在修改專案名稱時，發現將只單純修改大小寫並不會使 git 偵測到文件有被修改。

去網路上爬文才發現原來 git 本身對文件名是默認不區分大小寫的。

因此決定寫一篇文章將修改此設定的方法記錄下來。

其實這非常的簡單，只需要一行設定就行了。

# 取消設定忽略大小寫
git config core.ignorecase false

就這樣 ～ Done!
